# CLAUDE.md — 소상공인 지원금 큐레이션 사이트 (회사 매뉴얼)

이 파일은 AI 직원(마스터·기획·개발·디자인·CS·마케팅)이 이 프로젝트에서 일할 때 지키는 **회사 매뉴얼**이다.
회사를 운영하며 원칙을 계속 보완한다.

---

## 0. 목표 3줄 (채용 공고)

1. **목표:** 소상공인이 "내가 받을 수 있는 지원금/정책자금"을 5분 안에 찾게 해주는 사이트로 광고+제휴 수익을 만든다.
2. **매일 할 일:** 신규 공고 수집·정리·발행 → SEO 최적화 → "오늘의 지원금" 카드뉴스 1건 제작·업로드.
3. **결과물 방향:** 정부 포털(기업마당·정부24)보다 쉽고, 업종·지역별로 딱 맞는 정보 + "누가/얼마/언제까지/어떻게" 신청 가이드.

## 1. ⚠️ 검수 규칙 (가장 중요 — 거짓 완료 보고 금지)

모든 AI 직원과 마스터는 아래 2줄을 **무조건** 지킨다.

1. **결과물을 직접 열어 확인하고 증거를 제시할 것.** (배포 URL·생성된 파일 경로·API 응답 건수·스크린샷 등)
2. **확인 못 한 것은 "못 했다"고 솔직히 보고할 것.** 추측으로 "완료" 라고 하지 말 것.

> 예: "발행 완료"가 아니라 "dist/g/PBLN_...html 3개 생성 확인, 로컬 서버 200 응답 확인" 처럼 증거와 함께 보고.

## 2. 컴플라이언스 (법적 리스크 방지)

- 우리는 **순수 정보 제공 + 합법 제휴**만 한다. 정책자금 '대행/브로커' 행위 금지.
- 정부 기관을 사칭하지 않는다. 모든 페이지 하단에 **"본 사이트는 공식 정부 사이트가 아닙니다"** 고지 유지.
- 공고 원문을 그대로 전재하지 않고 **요약·재구성**한다. 출처(기업마당 등) 링크·표기.
- 리드폼으로 개인정보 수집 시 **개인정보처리방침** 필수.

## 3. 기술 스택 & 구조

| 영역 | 선택 |
|------|------|
| 정적 생성 | 의존성 0 순수 Node (`scripts/build.mjs`). 추후 Astro 이전 가능 |
| 데이터 | `data/announcements.json` (수집 결과), `data/sample.json` (키 없을 때 검증용) |
| 수집 | data.go.kr K-Startup 오픈API (`scripts/fetch.mjs`, `SERVICE_KEY` 필요). 엔드포인트 `apis.data.go.kr/B552735/kisedKstartupService01/getAnnouncementInformation01` |
| 배포 | `git push origin main` 자동 배포 (Netlify/Vercel 무료 티어). CLI 직접 배포 금지 |

**폴더 구조**
```
sbiz-jiwon/
  scripts/  lib.mjs(정규화·템플릿) · fetch.mjs(수집) · build.mjs(생성) · serve.mjs(로컬 확인)
  data/     sample.json · announcements.json(생성)
  dist/     생성된 정적 사이트(배포 대상)
```

## 4. 명령어

```bash
# 전체 경로로 node 실행 (node가 PATH에 없을 수 있음)
NODE="/c/Program Files/nodejs/node.exe"

# 1) 실데이터 수집 (.env 의 SERVICE_KEY 자동 로드) — 키 없으면 build가 sample.json 사용
"$NODE" scripts/fetch.mjs

# 2) 정적 사이트 생성 → dist/
"$NODE" scripts/build.mjs

# 3) 로컬 확인 (기본 4321 포트)
"$NODE" scripts/serve.mjs
```

## 5. 데이터 파이프라인 (매일 09:00 마스터가 지시)

1. 기업마당 API 호출 → 신규/변경 공고 수집 (분야: 금융·창업·경영 위주, 소상공인 관련 필터)
2. 각 공고를 "누가/얼마/언제까지/어떻게" 요약 + 업종·지역 태깅
3. `data/announcements.json`에 공고 ID로 upsert (중복 방지), 마감 지난 공고 아카이브
4. `build.mjs`로 정적 페이지 생성 (개별 공고 + 업종×지역 조합 + "이번 주 마감")
5. `git push` → 자동 배포
6. 결과 검증(§1 검수 규칙): 생성 파일 수·서버 응답 확인 후 보고

## 6. AI 직원 구성 (프로젝트 안에서만 일하도록 세팅)

- **마스터** — 매일 지시·검수·최종 승인. 산출물 직접 열어 검증.
- **기획팀** — 업종×지역 키워드 전략, 콘텐츠 우선순위.
- **개발팀** — API 수집·파싱·페이지 생성·버그 수정·배포.
- **디자인팀** — 카드뉴스 이미지, UI 개선.
- **CS팀** — 리드폼 문의 응답 초안, FAQ 관리.
- **마케팅팀** — 카드뉴스 문구, SNS 업로드 콘텐츠.

## 7. 작업 원칙 (요약)
